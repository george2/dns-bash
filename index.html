dns-bash

These are just a few DNS-related bash scripts I use to create a custom hosts file, among other things. They can be thought of in two ways: either as a set of DNS-backup scripts for the paranoid, or a way to significantly speed up browsing performance.

Hint: These were created when SOPA was under consideration.

How to use the scripts

addomains duckduckgo.com github.com www.example.com
# These are all the sites I want to have in my hosts file
# You can also just edit the 'domains' file manually, 
#  see domains.example for format.
optdomains
# This will make the domains file smaller and more efficient.
# It is optional, but recommended.
makedomains
# This is where the important stuff happens.
sudo cp hosts /etc/hosts
# If you're going to use this as your hosts file, install it in etc.

You can also leave the generated hosts file in the working dir, and use the other scripts (domgrep, domlookup, ldig) to do random stuff with domains and their IPs.

If you want to brute-force the internet, run findomains, wait a few hundred years until it finishes, then look in disappointment at the small portion of the internet for which you now have an already-out-of-date list of IPs and domains. Have fun with that.

Finally, if you want to test your setup to see what it would be like to live without dns, just run dnsinkhole as root. Press Ctrl-D or Ctrl-C to stop the sinkhole.
